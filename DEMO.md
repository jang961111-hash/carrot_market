## 🎯 테스트 시나리오

### 시나리오 1: 상품 둘러보기
1. 홈페이지에서 **6개 상품 카드** 확인
   - iPhone 13 Pro (85만원)
   - 에어팟 프로 2세대 (20만원)
   - 맥북 프로 M2 (150만원)
   - 나이키 운동화 (8만원)
   - IKEA 책상 (15만원)
   - 파이썬 프로그래밍 (1.5만원)

2. **카테고리 필터** 테스트
   - "전자제품" 클릭 → 3개 아이템
   - "의류" 클릭 → 1개 아이템
   - "ALL" 클릭 → 전체 표시

3. **상품 카드 클릭** → 상세 페이지 이동
   - 상품 이미지 (플레이스홀더)
   - 제목, 가격, 위치
   - 판매자 정보
   - 상품 설명

### 시나리오 2: 채팅하기
1. 상품 상세 페이지에서 **"💬 채팅하기"** 버튼 클릭
2. 자동으로 `/chat` 페이지로 이동
3. **좌측**: 채팅방 목록 (2개)
   - iPhone 13 Pro 채팅방
   - 맥북 프로 M2 채팅방
4. **우측**: 메시지 영역
   - 채팅방 선택 시 메시지 표시
   - 3개 기존 메시지 확인
5. **메시지 전송**
   - 입력창에 메시지 작성
   - Enter 또는 "전송" 버튼
   - 실시간 UI 업데이트 (Socket.io)

### 시나리오 3: 비디오 통화 (WebRTC)
1. 네비게이션에서 **"채팅"** 클릭
2. URL 직접 입력: `http://localhost:5173/call/1`
3. **카메라/마이크 권한** 허용
4. **"📞 전화 걸기"** 버튼 클릭
5. **Signaling 서버 연결 확인**
   - 개발자 도구 콘솔: "Connected to signaling server"
   - WebRTC 이벤트 로그

**참고**: 실제 통화는 2개 브라우저 또는 2개 기기 필요
